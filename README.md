# WP Visitor Stats

First-party **WordPress analytics** for logicencoder.com — page views, geo, technology, campaigns, custom events, IP bans, and a built-in URL shortener. No third-party tracker required for operator-owned metrics.

Private plugin: [logicencoder/wp-visitor-stats-plugin](https://github.com/logicencoder/wp-visitor-stats-plugin).

## The problem it solves

External analytics hide raw visitor data and complicate GDPR-style operator control. WP Visitor Stats keeps events in your database with an admin dashboard tuned for a single-site publisher.

## Tracking

JavaScript beacon (when enabled) plus optional server-side fallback on `wp`. Logs page views, scroll depth, page exit, custom events, UTM campaigns. Journey/click heatmap capture disabled in current JS build.

## URL shortener

Public redirects at **`{site}/go/{slug}`** — managed from admin with hit counts.

## IP bans

Banned addresses receive **403** before WordPress renders — useful for abuse patterns without a separate WAF rule.

## Admin dashboard

**Visitor Stats** menu: Overview, IPs, Live, Geo, Content, Technology, Campaigns, Custom Events, All Visitors, Ban List, URL Shortener, Diagnostics, Settings. CSV export and backup tools.

## Integrations

Tracking snippet can embed on static HTML generated by [mexc-live-stats-plugin](https://github.com/logicencoder/mexc-live-stats-plugin-overview) snapshot pages for unified analytics.
